# System Design Patterns

This project is a Java-based design patterns refresher for low-level design.

It is organized into 3 GoF categories:

- `creational`
- `structural`
- `behavioural`

## What is a Creational Pattern?

Creational design patterns deal with object creation.

They help us create objects in a way that is:

- flexible
- reusable
- easier to maintain

Instead of creating objects directly everywhere using `new`, creational patterns make object creation more controlled and better designed.

In simple words:

"Creational patterns focus on how objects are created."

## Why are creational patterns important?

They help when:

- object creation becomes complex
- one object should control how other objects are created
- the system should be loosely coupled
- object creation logic should be reused

## Examples of creational patterns

- Singleton
- Factory Method
- Abstract Factory
- Builder
- Prototype
